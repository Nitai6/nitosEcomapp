# Owner's 10 Blog Post Formulas (verbatim)

Pick one per post based on which fits the source material best.

| # | Formula | Best for |
|---|---|---|
| 1 | **Problem-Agitate-Solve** | Pain-point reframes + product as solution |
| 2 | **How-to step-by-step** | Tutorials, instructional |
| 3 | **List post** | "X ways to…", "Top X…", scan-friendly |
| 4 | **Before-and-after** | Transformation stories, social proof |
| 5 | **Question-and-answer** | Reddit/Quora source material |
| 6 | **Story-and-listen-and-take-away** | Narrative-led, emotional |
| 7 | **Comparison post** | "X vs Y", buying-stage searchers |
| 8 | **Mistake-to-fix** | "5 mistakes…", curiosity-gap |
| 9 | **Framework post** | Methodology, repeatable system |
| 10 | **Ultimate guide** | Pillar content, cornerstone, ≥4k words |

## Formula→length suggestions

- Problem-Agitate-Solve: 2,000–3,000
- How-to: 2,500–4,000
- List: 2,000–3,500
- Before-and-after: 2,000–3,000
- Q&A: 2,000–3,000 (one Q deep-dive) OR 4,000+ (multi-Q digest)
- Story: 2,500–4,000
- Comparison: 2,500–4,500
- Mistake-to-fix: 2,000–3,000
- Framework: 3,000–5,000
- Ultimate guide: 5,000–6,550 (cap)

## Selection logic

Agent picks via:
1. Source type → Reddit/Quora question = formula 5; tutorial source = formula 2; case study = formula 4 or 6
2. Search intent → informational broad = 10; commercial comparison = 7; transactional buying-mistake = 8
3. Existing pillars in topic plan → if pillar already exists, prefer narrower formulas (1, 3, 8) feeding into it
